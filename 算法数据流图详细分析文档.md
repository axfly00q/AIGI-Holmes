# AIGI-Holmes 项目算法设计与数据流图分析文档

## 项目概述

**AIGI-Holmes** 是一款面向新闻媒体核查场景的 AI 生成图片检测系统，采用微调的 ResNet-50 模型结合 CLIP 多模态分析，对图片进行真实性检测。本文档详细分析了项目的5大核心算法数据流，从多个维度展示了系统的架构设计和数据处理流程。

### 2️⃣ 单图检测算法数据流 (`02_单图检测算法数据流.png`)

**设计层次：** 单个图片检测算法的详细实现

**核心流程分解：**

#### 阶段1: 输入预处理
- **RGB转换与缩放**：将任意格式图片转为 RGB，缩放至 224×224
- **张量化与归一化**：使用 ImageNet 标准化参数（mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225])
- **输出**：Tensor shape (1, 3, 224, 224)

**关键代码位置：** [backend/detect.py](backend/detect.py) 中的 `_transform` 变量和 `detect_image()` 函数

#### 阶段2: 深度学习推理
- **ResNet50主干网络**：预训练权重微调，包含 conv1 → layer1/2/3/4
- **中间特征提取**：layer4 输出提取 (1, 2048, 7, 7) 维特征
- **全连接层**：2048 维 → 2 维（对应 FAKE/REAL 两类）

**关键代码位置：** [detect.py](detect.py) 中模型加载的 `_load_model()` 函数

#### 阶段3: 结果获取
- **Softmax概率计算**：(1, 2) logits → 概率归一化
- **最终分类**：选取置信度最高的类别
- **规则化解释**：基于置信度分级（极高/高/中等/低置信度）

**关键代码位置：** [detect.py](detect.py) 中的 `explain_result()` 函数

#### 阶段4: Grad-CAM 可解释性分析（可选）
Grad-CAM（梯度加权类激活映射）过程：

1. **收集梯度**：对 layer4 注册前/后向钩子，捕获梯度
2. **计算权重**：$W = \frac{\partial L}{\partial A}$ （权重 = 梯度按通道求平均）
3. **加权求和**：$CAM = \sum w_c A_c$（加权求和得特征图）
4. **ReLU激活**：$\max(CAM, 0)$（保留正激活）
5. **双线性插值**：将 7×7 特征图插值回 224×224 原始尺寸

**输出**：Red channel = 激活强度，Alpha channel = 透明度 → Base64 JPEG 编码

**关键代码位置：** [detect.py](detect.py) 中的 `_grad_cam()` 和 `grad_cam_overlay()` 函数

#### 阶段5: 结构化输出
```python
{
    "label": "FAKE" | "REAL",
    "label_zh": "🤖 AI生成" | "📷 真实照片",
    "confidence": 45.2,  # 百分比 (0-100)
    "probs": [
        {"label": "FAKE", "label_zh": "🤖 AI生成", "score": 45.2},
        {"label": "REAL", "label_zh": "📷 真实照片", "score": 54.8}
    ],
    "explanation": {
        "level": "中等置信度",
        "summary": "...",
        "clues": ["人物手指异常", "背景纹理过规整", ...],
        "disclaimer": "结果仅供参考..."
    },
    "cam_image": "data:image/jpeg;base64,..."  # 仅当请求 cam=1 时
}
```

### 3️⃣ CLIP多模态分析数据流 (`03_CLIP多模态分析数据流.png`)

**设计层次：** 多模态深度学习算法的双分支流程

**模型选择：** OpenAI CLIP ViT-B/32（512维特征空间）

#### 左分支：内容分类流程
**用途**：对图片进行 7 类内容分类（人物/动物/建筑/风景/食物/交通/其他）

**算法步骤：**
1. **图像编码**：通过 CLIP Vision Transformer 编码器
   - 输入：PIL Image → 预处理 → Patch Embedding
   - 输出：512维单位向量 feature_img
   
2. **文本编码**（一次性，缓存复用）：
   - 7个提示词：
     - "a photo of a person or people"
     - "a photo of an animal"
     - "a photo of buildings or architecture"
     - ... 等7个
   - 通过文本编码器 → 7×512维特征矩阵
   - 所有特征 L2 归一化

3. **余弦相似度计算**：
   ```python
   sim = img_features @ text_features.T  # (1, 512) @ (512, 7) → (1, 7)
   ```

4. **分类结果**：
   ```python
   category_idx = argmax(similarities)
   category_label = CATEGORIES[category_idx]  # 返回中文标签
   ```

**关键代码**：[backend/clip_classify.py](backend/clip_classify.py) 中的 `classify_image()` 函数

**性能特点：**
- ✓ 零样本学习（zero-shot），无需训练
- ✓ 首次加载 CLIP ~350MB，后续复用
- ✓ 文本特征预计算，推理时间短

#### 右分支：图文一致性检测流程
**用途**：评估图片与新闻文本的匹配程度

**关键创新：** 基准线方法（baseline method）消除单一提示词方差

**算法步骤：**
1. **图像编码**：同左分支
   
2. **双层提示词策略**（共7个）：
   - **对齐提示词（4个）**：与用户文本相关
     - "a photo showing: {text}"
     - "an image related to: {text}"
     - "a picture illustrating: {text}"
     - "a photo matching the description: {text}"
   - **不对齐基准提示词（3个）**：与文本无关
     - "a random photo unrelated to the topic"
     - "a photo of something completely different"
     - "an abstract or unrelated image"

3. **相似度计算**：
   ```python
   sims_aligned = similarities[0:4].mean()     # 对齐提示词平均
   sims_unrelated = similarities[4:7].mean()   # 不对齐提示词平均（基准）
   gap = sims_aligned - sims_unrelated         # 相似度间隙
   ```

4. **间隙归一化为评分**：
   ```python
   # 将 gap ∈ [-0.20, +0.20] 映射至 [0, 100]
   normalized = max(0.0, min(1.0, (gap + 0.15) / 0.30))
   score = round(normalized * 100, 1)
   ```

5. **评级划分**：
   ```
   score >= 65  → "高度一致"（match_score ≥ 65）
   42 <= score < 65 → "基本一致"
   score < 42   → "不一致"
   ```

**关键代码**：[backend/clip_classify.py](backend/clip_classify.py) 中的 `classify_text_image_consistency()` 函数

**返回结构：**
```python
{
    "score": 72.5,  # 0-100
    "assessment": "高度一致",
    "details": "图片内容与文本描述高度匹配，图文关联性强。"
}
```

### 4️⃣ URL批量检测与实时进度推送数据流 (`04_URL批量检测与实时进度推送.png`)

**设计层次：** 分布式异步处理与实时反馈系统

**场景**：用户输入新闻页面 URL → 自动爬取图片 → 并发检测 → WebSocket 实时推送进度

#### 第1层：用户请求
- 输入：新闻页面 URL
- 执行：`POST /api/detect-url`（初始化检测任务）

#### 第2层：页面爬取与图片收集
**关键步骤：**
1. **URL验证**：SSRF安全防护（private IP 黑名单）
   ```python
   validate_public_url(url)  # 拒绝 127.0.0.1, 192.168.*, 10.0.0.0, 等
   ```

2. **页面下载**：HTTP 请求获取 HTML 内容

3. **HTML解析**：使用 BeautifulSoup 提取 `<img>` 标签

4. **图片URL提取**：收集所有 src 属性，转为绝对 URL

5. **特征对比**：通过 SHA-256 哈希检测重复图片

**相关代码**：[detect.py](detect.py) 中的 `async_fetch_page_content()` 和 `async_fetch_image_urls()` 函数

#### 第3层：异步并发处理
**架构**：asyncio 事件循环 + Queue

**关键特性：**
- 最多 5 个并发下载流：`asyncio.Semaphore(5)`
- 内存缓冲编制：asyncio.Queue
- 逐张下载到内存，形成 [img1, img2, ..., imgN]

**关键代码**：[detect.py](detect.py) 中的 `async_download_image()` 和 `asyncio.gather()` 的并发调用

#### 第4层：检测推理与缓存
**三级优化**：

1. **Redis缓存检查**：
   - Key: `detect:{image_sha256}`
   - 避免重复推理整个相同图片
   ```python
   cached = await get_cached_result(raw_bytes)
   if cached: return cached
   ```

2. **批量推理**：`detect_batch(images)`
   - 与单图检测相同的模型，但利用 GPU 批处理优势
   - 单次 forward pass → N 个结果
   - 代码：[detect.py](detect.py) 中的 `detect_batch()` 函数

3. **CLIP分类与一致性检测**：
   - 对每张图片执行 `classify_image()`
   - 如果有图片描述文本，执行 `classify_text_image_consistency()`

#### 第5层：WebSocket 实时反馈
**推送机制**：

事件流：
```json
// 进度中间事件
{
    "type": "progress",
    "index": 3,
    "total": 20,
    "result": {
        "label": "FAKE",
        "confidence": 78.5,
        "thumbnail": "base64://...",
        "category": "建筑",
        "consistency": {"score": 65, "assessment": "高度一致"}
    }
}

// 完成事件
{
    "type": "complete",
    "results": [...],
    "overall_score": 72.1,
    "dimensions": {
        "fake_ratio": 0.35,
        "avg_confidence": 72.1,
        ...
    }
}
```

**前端表现**：
- 实时进度条：3/20
- 缩略图+标签+置信度
- 无需轮询，推送延迟 <100ms

**相关代码**：[backend/routers/ws.py](backend/routers/ws.py) 中的 `ws_detect_progress()` 函数

### 5️⃣ 检测报告生成与多格式导出工作流 (`05_检测报告生成与多格式导出.png`)

**设计层次：** 数据聚合与多渠道输出系统

**场景**：用户点击"生成报告" → PDF/Excel/JSON 多格式导出

#### 第1阶段：检测数据聚合
**数据源汇总：**
- 检测ID (`detection_id`)
- 用户信息 (`user_id`)
- 检测结果标签与置信度 (`label`, `confidence`)
- CLIP内容分类 (`category`)
- 图文一致性评分 (`consistency`)
- Grad-CAM热力图（Base64编码）

#### 第2阶段：数据库查询与聚合
**函数**：`generate_report(detection_id) → dict`

**SQL查询**：
```sql
SELECT * FROM detection_records 
WHERE id = ? 
LEFT JOIN users ON detection_records.user_id = users.id
LEFT JOIN report_metadata ON detection_records.id = report_metadata.detection_id
```

**数据聚合**：合并来自多张表的信息为单一 report_dict

#### 第3阶段：结构化报告生成
**报告组成模块**：
1. **基本信息**：检测ID、用户、时间戳
2. **检测结果**：标签、置信度、概率分布
3. **热力图**：Grad-CAM 可视化
4. **内容分类**：CLIP 7类分布
5. **一致性评分**：图文匹配度
6. **AI分析**：豆包 AI 自然语言总结
7. **审计日志**：检测时间、版本号等

**结构**：
```python
report_dict = {
    "metadata": {...},
    "detection": {
        "label": "FAKE",
        "confidence": 78.5,
        "probs": [...]
    },
    "clip_analysis": {
        "category": "建筑",
        "consistency": {...}
    },
    "visualizations": {
        "cam_image": "data:image/jpeg;base64,..."
    },
    "ai_analysis": "豆包AI生成的分析文本..."
}
```

#### 第4阶段：多格式导出
**导出三个分支**：

##### PDF导出
- **库**：reportlab / WeasyPrint
- **特性**：
  - 水印加载（"AI检测报告"斜体）
  - 检测结果横幅（FAKE 红色框，REAL 绿色框）
  - 嵌入热力图
  - A4 规范排版
- **函数**：`export_pdf(report_dict) → bytes`

##### Excel 导出
- **库**：openpyxl
- **特性**：
  - 多 Sheet（基本信息、检测结果、分析细节）
  - 数据表格（便于统计分析）
  - 置信度柱状图
  - 冻结首行
- **函数**：`export_excel(report_dict) → bytes`

##### JSON 导出
- **特性**：
  - 完全结构化
  - 机器可读
  - 便于 API 集成
  - indent=2 格式化
- **函数**：内置，直接返回 JSON

#### 第5阶段：文件输出与缓存
- PDF → report_ID.pdf（字节流存入 /tmp）
- Excel → report_ID.xlsx
- JSON → report_ID.json
- **缓存策略**：Redis 存储 60 分钟，避免重复生成

#### 第6阶段：HTTP 响应与下载
**HTTP 响应头**：
```http
HTTP/1.1 200 OK
Content-Type: application/pdf | application/vnd.openxmlformats-officedocument.spreadsheetml.sheet | application/json
Content-Disposition: attachment; filename=report_ID.pdf
Content-Length: 123456
```

**前端行为**：浏览器自动触发下载

**相关代码**：
- [backend/routers/report.py](backend/routers/report.py)（路由）
- [backend/report/generator.py](backend/report/generator.py)（生成逻辑）
- [backend/report/exporter.py](backend/report/exporter.py)（导出实现）

## 🎯 核心算法总结表

| 算法 | 功能 | 模型 | 输入 | 输出 | 性能 |
|------|------|------|------|------|------|
| **单图检测** | FAKE/REAL 二分类 | ResNet-50微调 | 图片(任意尺寸) | 标签+置信度 | ~50ms/图(GPU) |
| **Grad-CAM** | 可解释性热力图 | Layer4梯度钩子 | 图片+类别索引 | Base64热力图 | ~100ms/图 |
| **内容分类** | 7类内容识别 | CLIP ViT-B/32 | 图片 | 类别标签 | ~80ms/图(首次) |
| **图文一致性** | 图片与文本匹配度 | CLIP ViT-B/32 | 图片+文本 | 评分[0-100] | ~120ms/对(首次) |
| **URL爬取** | 网页图片批量收集 | BeautifulSoup | URL | 图片列表 | ~1-5s(取决于网络) |
| **批量推理** | 多图并发检测 | ResNet-50 | 图片列表 | 结果列表 | ~10ms/100图(GPU) |

## 🔄 典型工作流示例

### 场景1：用户上传单张图片检测
```

    ↓

    ↓

    ↓ (缓存命中)

    ↓ (缓存未命中)

    ↓

    ↓

    ↓

    ↓

    ↓
```

## 📈 性能指标与优化

### 推理速度（GPU: NVIDIA RTX3080）
- **单图检测**：~50ms
- **Grad-CAM**：+100ms
- **CLIP分类**：~80ms（首次加载 CLIP ~2s）
- **图文一致性**：~120ms（复用文本特征）
- **批量100图**：~500ms（相对於单张 50 个）

### 内存占用
- **ResNet50模型**：~100MB
- **CLIP模型**：~350MB
- **缓存系统**：取决于检测量，建议 Redis 2GB

### 并发能力
- **URL爬取**：Semaphore=5 限制TCP连接
- **模型推理**：单 GPU 支持 batch_size=64
- **WebSocket连接**：asyncio 支持数千并发

2. **多维分析**：结合 ResNet-50、CLIP、Grad-CAM，从多角度判断
3. **实时反馈**：WebSocket 实时推送，无需轮询
4. **多格式导出**：PDF、Excel、JSON，满足不同需求
5. **缓存复用**：Redis 缓存 + 批量推理，大幅降低重复计算
