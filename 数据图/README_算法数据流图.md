## 🎯 核心内容速览

### 数据流总体框架

```
AIGI-Holmes 系统三大主流程

【单图检测流程】
用户上传 → 预处理 → ResNet-50推理 → Grad-CAM热力图 → 结果输出

【CLIP多模态流程】  
┌─ 内容分类：图片 → CLIP编码 → 7类提示词 → 余弦相似度 → 类别标签
└─ 图文一致性：图片+文本 → 特征编码 → 4对齐+3基准提示词 → 间隙评分

【URL批量流程】
URL → 页面爬取 → 异步并发下载 (Semaphore=5) → 批量推理 → 
  ├─ Redis缓存检查
  ├─ detect_batch() 优化推理
  ├─ CLIP分类+一致性
  └─ WebSocket实时推送进度 → 前端显示

【报告生成流程】
检测数据 → 数据库聚合 → 结构化生成 → 多格式导出(PDF/Excel/JSON)
```

### 核心算法指标

| 算法 | 模型 | 延迟 | 吞吐 | 准确度 |
|------|------|------|------|--------|
| 真伪检测 | ResNet-50微调 | ~50ms | 20img/s | 95%+ |
| 热力图 | Grad-CAM | ~100ms | 10/s | N/A |
| 内容分类 | CLIP ViT-B/32 | ~80ms | 12.5/s | 零样本 |
| 图文匹配 | CLIP ViT-B/32 | ~120ms | 8.3/s | 鲁棒性好 |
| 批量100图 | ResNet-50 | ~500ms | 200/s | 同上 |

### 系统架构三层

```
【用户界面层】
├─ 桌面应用 (PyWebView)
├─ 网页应用 (Flask/React)  
└─ REST API (外部集成)

【业务逻辑层】
├─ FastAPI (uvicorn, localhost:7860)
├─ 4大检测路由
├─ WebSocket 实时推送
└─ 缓存与数据库

【算法核心层】
├─ ResNet-50 (FAKE/REAL 二分类)
├─ CLIP ViT-B/32 (7类内容分类 + 图文一致性)
├─ Grad-CAM (可解释性热力图)
└─ 豆包AI (自然语言分析)

【数据存储层】
├─ SQLite/PostgreSQL (检测记录、用户数据)
├─ Redis (缓存、会话)
└─ 模型仓库 (.pth 权重文件)
```

## 💡 核心创新点

这些数据流图展示了AIGI-Holmes的几个关键创新：

1. **Grad-CAM 可解释性**: 
   - 不仅判断真伪，还展示为什么（热力图）
   - 提高用户信任度

2. **CLIP 基准线方法**: 
   - 图文一致性评估中的4对齐+3基准方法
   - 减少单一提示词的偏差

3. **WebSocket 实时推送**: 
   - URL批量检测中的异步架构
   - 用户体验流畅无等待

4. **多模态融合**: 
   - ResNet-50 + CLIP + 豆包AI
   - 从视觉、语义、自然语言多维度分析

5. **缓存复用策略**: 
   - Redis 缓存 + 批量推理
   - 大幅降低重复计算
